第六屆wehelp軟體工程師轉職計畫

Q1.個人簡介與申請動機

•背景：護理系畢業，反思護理工作的生活方式，決定尋找符合自身期望的職涯發展。

•動機：接觸網路技術和程式設計後產生濃厚興趣，在之前的工作經驗具有創新、抗壓性、學習力強、團隊溝通、解決問題能力的人格特質。

我是詹力臻，今年 26 歲，畢業於長庚科技大學護理系。畢業後，我進入醫療體系，現任職於新北市立土城醫院。在工作中，我展現出樂觀開朗的個性，即使遇到困難也會積極面對，並從中學習與成長。每次成功解決問題後，能感受到自我的進步，這讓我充滿成就感。

工作中，我需要面對各種病人及突發狀況。為了解決問題，我常與同事及主管合作，討論最佳解決方案，以便在未來類似情況下能快速反應並應用經驗進行改善。特別是在疫情爆發期間，我被派往專責病房，與團隊合作規劃最佳進出動線與標準處理流程，並引入 QR CODE 技術協助病患溝通。當時，由於防疫政策的不斷變化，我們持續優化病房的工作守則，確保患者與醫護人員的安全。這段經歷讓我深刻體會到團隊合作與問題解決的重要性，也累積了寶貴的應變能力。

如今，我已能勝任護理工作，並在不同科別中獨立完成任務。然而，疫情的經歷讓我重新審視自己的職涯方向。我發現目前的工作生活模式與未來期待有所差距，因此開始探索其他職涯可能性。我列出自己感興趣、擅長且願意投入時間的事物，並逐步學習相關知識。在此過程中，我自學了網路基礎概論、CCNA、程式設計等內容，並發現程式開發的成就感深深吸引了我。

透過基礎學習，我逐漸了解網路世界背後的運作，以及網頁與資料庫如何因應使用者需求靈活調整。能夠將腦中的想法實現為具體作品，讓我感到前所未有的滿足與興趣。我也認識到程式開發對未來的巨大影響，這激發了我深入學習的渴望，並相信這是一個提升自我、實現職業發展的絕佳機會。

Q2.曾經做過哪些軟體工程技術相關的學習？若有作品請分享給我們。

•自學經歷：網路基礎概論、Python、HTML、JavaScript、Flask、MongoDB 和 GitHub。

在 YouTube 上，我接觸到 Papaya 電腦教室的基礎 Python 程式教學，激發了對程式設計的興趣。隨後，我透過搜尋找到更多線上學習資源，包括 Udemy 和彭彭的課程，進一步學習撰寫程式。在學習過程中，我會跟隨教學逐步完成專案，同時在程式碼旁註解原因，幫助理解程式邏輯，並將筆記整理成系統化文件，以方便未來查詢與複習。此外，當我有新的想法時，會試著採用不同的方法來實作，並添加創新的元素，進行反覆嘗試與創意激盪，最終成功執行程式，這段過程讓我充滿成就感。

目前，我已掌握 Python 基礎、HTML、JavaScript、Flask、MongoDB 和 GitHub 的基本應用。這些技能使我能夠完成一系列基礎專案，並對程式開發的流程與架構有了更清晰的認識。

Q3.如果參與這個訓練，會怎麼安排學習時間？

由於目前仍在職，若能有幸參與訓練，預計會在領取年終獎金後辭職，並全心投入網站開發課程的學習。離職前，我將妥善安排時間參與每週一上午 10 點的線上會議，並利用下班後的時間完成課程學習及每週任務，計劃於 1 月 20 日前完成離職手續。

課程期間，我將安排固定的學習計劃：每週一至週五上午 10 點到下午 6 點集中精力完成任務進度。如果遇到困難，會持續解決問題直到進度完成；若提前完成，則利用剩餘時間學習其他相關技術。週六將用來確保所有任務順利達成，週日則給予自己一小段休息時間，並系統化整理與複習一週的學習內容。

Q4.請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

•應對負面情緒：接受自己不足之處，但不輕言放棄，自我砥礪。

畢業後投入職場，進入了一間新開幕的醫院，由於人手不足，原本三個月的新人期被縮短為一個月，需在短時間內獨立上線。那段期間，我經歷了長達半年的撞牆期。在校所學與職場上需要的能力有很大的落差，許多事情都需要重新摸索和學習，包括熟悉工作規範、操作技術、學姊在臨床上教導的技術、檢查準備、以及與病人的應對溝通。

壓力對我來說非常大，因為犯錯可能會影響生命，甚至一個家庭。因此，我試圖在短時間內克服這些挑戰，將工作規範整理成筆記，利用下班後的時間熟練技術，並整理一本專屬的工作筆記。即使如此，上線後，我仍遇到許多以前從未碰過的情況，例如病人狀況瞬息萬變、家屬的提問等，這些都需要依賴我之前所學，或查閱醫院的資料庫來解答。若無法解決，我便尋求學姊或醫師的協助並從中學習。

在職場上，我也免不了遇到前輩的刁難和閒言閒語，當下聽到雖然內心感到難過。但由於工作時間有限，我只能先專注於解決問題，不會就問，問完學起來。回家後整理思緒，偶爾會情緒低落，懷疑自己是否適合這份工作。不過，我告訴自己，畢業時已經在學校學了四年，工作才不到一年，我不甘心在剛開始就放棄自己，還未竭盡全力就放棄，未來只會徒留遺憾。

我意識到前輩們擁有豐富的工作經驗，我還在成長的過程中，只要能抓住做事的訣竅，我也能慢慢適應。工作速度慢可以練習，不會的技術多做幾次，不會說話應對那就多聽多學，慢慢就會進步。我告訴自己，我現在像是一棵小樹苗，雖然成長的速度可能不快，但只要找到屬於自己的「專屬時區」，終將會茁壯。

Q5.關於這份申請網頁，分享一個開發時的技術心得

•開發過程：雖有程式基礎，但在實際製作時仍感到挑戰，規劃內容結構，遇到問題時，善用工具尋求解答。

雖然我曾學過基礎的 HTML 和 CSS，但在開始製作個人網站時，腦中一片空白，無法立刻構想出希望的網頁格式。於是，我先將所有回覆內容整理出來，作為網站製作的基礎。第一步，我使用div標籤將內容分成一塊一塊，逐步查看在網頁上的呈現效果。接著，我開始規劃字體顏色、邊距、行高等細節，並將每個問題的答覆設計成區塊格式。

在設計過程中，我首先為第一個答覆區塊編寫class標籤，設定其位置、背景顏色、文字樣式與區塊邊距等細節。只要規劃好想呈現的方式，後續每個答覆都可以套用相同的模板，提升開發效率。

在開發過程中，遇到想實現某些效果卻不知道具體指令時，我會使用 ChatGPT 尋求解答。此外，在撰寫程式時，我常嘗試調整各種設定，例如圓角邊框的大小，以觀察不同的呈現效果。當看到網頁變得整齊、美觀且統一時，我感到非常有成就感，這也激勵我不斷改進自己的設計與編程能力。

Q6.如何看待自身工作和整個社會群體的連結關係？

•科技蓬勃發展為生活帶來巨大影響，希望結合護理專業開發健康教育或醫療輔助產品，提升大眾生活品質。

目前我從事護理行業，社會大眾對這份工作的印象通常侷限於「照護病人、打針、發藥」這類日常任務。然而，實際投入這個行業後，我深刻體會到，護理工作遠不止於此。很多時候，民眾對於醫療健康的知識不足，加上網路上流傳的錯誤資訊或過時的觀念，往往會讓人在身體發出警訊時做出錯誤的選擇，甚至導致不可逆的健康損害。因此，我認為我的工作不僅僅是幫助病人康復出院，更重要的是向他們傳遞正確的健康觀念與預防保健的知識。無論是幫助慢性病患者維持穩定的健康狀況，還是協助延緩疾病的進展，這些教育工作都能為社會帶來深遠的正面影響。

隨著科技的日益進步，對我們的生活產生了越來越深遠的影響。如今，大多數人會依賴 Google 或 ChatGPT等工具尋找答案，使用電商網站購物，這些科技正在改變人們的行為模式與生活方式。希望能加入課程並成功畢業，進一步成為一名初階軟體工程師，像公司中的小螺絲釘一樣，穩紮穩打地為團隊與工作貢獻自己的力量。同時，也希望在工作中不斷學習與成長，進一步對這份職業有更多理解，並尋找屬於自己的價值。之後若能成為獨當一面軟體工程師，也希望將護理專業與科技兩項專業結合，開發出以健康教育或醫療輔助為核心的產品，讓大眾獲得更準確且便利的健康資訊，從根本上幫助他們改善生活品質。我相信，隨著經驗的累積，未來對軟體工程師這份工作的看法和目標也會不斷深化，甚至帶來更多全新的可能性。
